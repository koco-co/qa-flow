# PRD-15530 【内置规则丰富】合理性校验-多表字段值对比（计算逻辑）

> 来源：蓝湖原型 15530 + 源码分析（`release_6.3.x_ltqc`）
> 模块：数据资产 > 数据质量 > 规则配置
> 开发版本：6.3岚图定制化分支
> 功能状态：全新需求（源码现有规则最高 ID=44）
> 增强日期：2026-03-29

---

## 1. 程序描述

在现有「合理性校验」规则类型下，新增**多表字段值对比**子类型，支持：
1. 配置多张关联表（最多 10 张）及主键关联关系
2. 为校验字段编写「计算逻辑」（支持跨表字段运算，如 `表a.test_col_2 + 表b.test_col_3`）
3. 两种对比方法：**计算结果对比**（两段逻辑互比）/ **计算结果值判断**（结果与固定值比）
4. 配置过滤条件（前置过滤被校验数据）
5. 查看校验不通过的明细数据（含多表字段列）
6. 在质量报告中展示新规则的质检结果

**数据类型约束（重要）：**
- 仅支持数值型：TINYINT、SMALLINT、MEDIUMINT、INT/INTEGER、BIGINT、FLOAT、DOUBLE/DOUBLE PRECISION、DECIMAL/NUMERIC
- Hive/Spark/Doris 数据源中 string 类型字段需**强转为 double 类型**
- 字段选择器中**只展示**上述数值型字段

## 3. 流程逻辑

### 3.1 规则配置入口

```
数据质量 > 规则配置 > 新建规则
  → 规则类型 = 合理性校验
  → 合理性类型 = 多表字段值对比
  → 配置规则字段（字段/计算逻辑/对比方法/关联表/主键/过滤条件）
  → 保存规则
```

### 3.2 规则执行结果查看

```
任务实例列表 → 选择实例 → 查看规则列表
  → 若"校验不通过" → 点击【查看详情】→ 弹窗展示明细数据
  → 若"校验通过" → 不记录明细数据
  → 若"校验失败" → 点击【查看日志】
```

## 4. 功能详细设计

### 4.1 规则配置页面

#### 4.1.1 合理性校验-多表字段值对比（计算结果对比模式）

##### 页面说明

原型展示：合理性校验配置表单，对比方法选择「计算结果对比」时，配置两段计算逻辑进行互相比较。

##### 字段定义（规则表单）

| 字段名称 | UI 标签 | 字段类型 | 是否必填 | 枚举值 / 校验规则 |
|---------|---------|---------|---------|-----------------|
| 校验字段 | * 字段 | 下拉选择 | 是 | 仅展示数值型；string 类型需加强转（源码约束） |
| 计算逻辑1 | * 计算逻辑配置 | 输入框（带字段选择器） | 是 | 支持点击字段自动填充；支持手动输入运算符（+/-/*//** 等） |
| 对比方法 | * 对比方法 | 下拉选择 | 是 | `>`/`>=`/`<`/`<=`/`=`/`!=` |
| 比较方式 | （无显式标签） | 单选 | 是 | 计算结果对比 / 计算结果值判断 |
| 关联表（N=1..10） | * 关联表N | 数据表选择器 | 是 | 最多添加 10 个关联表 |
| 关联表主键 | * 关联表N主键 | 下拉选择 | 是 | 悬浮提示：根据主键关联一致的数据进行计算校验 |
| 校验表主键 | * 校验表主键 | 下拉选择 | 是 | 当前校验表的主键字段 |
| 计算逻辑2 | * 计算逻辑配置2 | 输入框（带字段选择器） | 是（仅计算结果对比模式） | 固定展示长度，超出鼠标悬浮展示全部 |

##### 字段定义（过滤条件）

| 字段名称 | UI 标签 | 字段类型 | 是否必填 | 枚举值 / 校验规则 |
|---------|---------|---------|---------|-----------------|
| 过滤字段 | 字段 | 下拉选择 | 否 | 如 `test_col_1` |
| 比较方式 | — | 下拉选择 | 否 | 固定值 / 字段 |
| 比较符 | — | 下拉选择 | 否 | `<`/`>`/`=`/`!=`/`>=`/`<=` |
| 固定值 | — | 输入框 | 否 | 如 `1` |

##### 交互规则

- 规则类型选「合理性校验」后，下方出现「合理性类型」子选项，选「多表字段值对比」后展示本节表单
- 「计算逻辑配置」编辑框：
  - 点击字段名**自动填充**到编辑框（可选当前所有关联表下的字段）
  - 字段格式：`表a.字段名`（表别名由系统自动分配，如 a/b/c）
  - 支持手动输入运算符，直接拼接到 SQL
- 「比较方式」切换时，表单项动态变化：
  - 计算结果对比：出现「计算逻辑配置2」；无「结果值比较符」
  - 计算结果值判断：无「计算逻辑配置2」；出现「结果值」比较符和固定值输入
- 关联表按钮：点击【添加关联表】增加一行；最多添加 10 个；可删除

#### 4.1.2 合理性校验-多表字段值对比（计算结果值判断模式）

##### 页面说明

原型展示：对比方法选择「计算结果值判断」时，将计算逻辑的结果与一个固定数值进行比较。

##### 字段定义（与模式切换相关的差异字段）

| 字段名称 | UI 标签 | 字段类型 | 是否必填 | 枚举值 / 校验规则 |
|---------|---------|---------|---------|-----------------|
| 结果值比较符 | 结果值 | 下拉选择 | 是 | `>`/`<`/`=`/`!=`/`>=`/`<=` |
| 关联表N所属库 | * 关联表N所属库 | 下拉选择 | 是 | 关联表所属数据库 |
| 固定比较值 | （结果值输入框） | 数字输入 | 是 | 数值型 |

> ⚠️ [W002] 原型中「关联表N所属库」在「计算结果值判断」模式下显式出现，但在「计算结果对比」模式中未明确说明是否存在，建议与开发确认是否两种模式均需填写。

##### 交互规则

- 默认选择当前校验表，支持选择其他关联表
- 仅支持数值型（含 string 强转）校验

### 4.2 任务实例-结果明细弹窗

#### 4.2.1 查看入口

- 路径：任务实例 > 规则执行结果列表 > 「查看详情」（仅校验不通过时可点击）
- 弹窗标题：**查看「合理性校验-多表字段值对比」明细**

#### 4.2.2 明细数据列定义

| 列名格式 | 说明 |
|---------|------|
| 校验表.主键字段 | 主键，支持多主键多选；支持复制表名 |
| 校验表.校验字段 | 不符合规则的数据**标红**展示 |
| 校验表.计算字段N | 参与计算逻辑的校验表字段 |
| 关联表M.主键字段 | 关联表 M 的主键列 |
| 关联表M.字段K | 关联表 M 参与计算逻辑的字段 |

##### 交互规则

- 校验通过：**不记录明细数据**（无查看入口）
- 校验不通过：记录不符合计算逻辑的数据行
- 校验失败：展示【查看日志】按钮
- 主键支持多选
- 支持复制表名
- 后端需解析计算逻辑配置的字段信息并存储在明细数据表中（多表规则特殊处理）

## 6. 测试重点

1. **两种对比模式**（计算结果对比 vs 计算结果值判断）的配置表单差异、保存与执行
2. **数值类型约束**：非数值型字段无法选中；Hive/Spark/Doris string 类型强转
3. **关联表上限**：最多 10 个关联表的边界测试
4. **计算逻辑编辑框**：字段自动填充、手动输入运算符、表别名格式（`表a.字段`）
5. **过滤条件**：有/无过滤条件时的执行行为差异
6. **明细数据**：通过时不记录；不通过时明细数据列动态生成（含多表字段）
7. **质量报告**：详情说明文案的正确拼接（两种模式格式不同）
8. **规则库**：规则分类与描述是否正确展示
