# xyzh 项目用例编写偏好

> 优先级：用户当前指令 > 项目级偏好规则 > 全局偏好规则 > skill 内置规则

## 适用范围

- 项目：`xyzh`
- 适用场景：项目缺少源码和知识库，测试用例主要依据 PRD 或现有原型信息编写

## 编写规则

1. 用例采用**概括性写法**，以 PRD 中可确认的页面、流程、按钮、字段、状态和联动关系为主。
2. 前置条件只写**必要业务状态**和**页面访问条件**，不扩写为冗长的造数脚本或虚构初始化步骤。
3. 禁止编造或写死以下占位数据：
   - 账号名、工号、人员姓名
   - 表名、API 名、字段名、字段值、示例参数值
   - 具体时间、同步时间、排序首条、结果数量
   - 文件名、附件名、目录路径、资源路径
4. 若 PRD 未提供明确示例，步骤中使用“输入关键字”“选择目标资源”“填写必填信息”“准备超限文件”等可执行表述，不补充虚构样例值。
5. 预期结果优先描述可观察的页面行为，包括：
   - 控件/字段/Tab/按钮是否展示
   - 查询、筛选、排序、分页是否生效
   - 发布、上架、下架、同步、保存后的状态变化
   - 前后台联动是否生效
6. 当需要验证数据差异时，写“展示已发布版本/最新发布版本/后台已维护内容”等相对表达，不写死具体文案内容，除非 PRD 明确给出。
