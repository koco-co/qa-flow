# 【内置规则丰富】有效性，json中key对应的value值格式校验

## 需求概述

| 项目     | 内容                                                                 |
| -------- | -------------------------------------------------------------------- |
| 开发版本 | 6.3岚图定制化分支（数据资产V6.4.10） 🟢                              |
| 需求背景 | 有效性校验支持对json类型的字段做key对应的value值格式的校验 🟢        |
| 影响模块 | 数据质量 → 规则配置、结果查询、规则库 🟢                             |
| 导航路径 | 数据质量 → 数据校验 → 规则配置 / 结果查询 🔵 `/dq/rule`              |
| 关联需求 | 依赖 #15696【通用配置】json格式配置（提供key列表及value格式定义） 🟢 |

## 页面：规则配置 - 新增"格式-json格式校验"规则

> 📍 导航路径：数据质量 → 数据校验 → 规则配置 → 添加/编辑规则
> 📸 来源：蓝湖 #15694

### 规则基本信息

| 字段名   | 值                | 来源 |
| -------- | ----------------- | ---- |
| 规则类型 | 字段级            | 🟢   |
| 规则名称 | 格式-json格式校验 | 🟢   |
| 规则分类 | 有效性校验        | 🟢   |
| 关联范围 | 字段              | 🟢   |

### 字段定义 — 格式-json格式校验配置

| 字段名   | 控件类型       | 必填 | 校验规则                                                  | 默认值 | 来源 |
| -------- | -------------- | ---- | --------------------------------------------------------- | ------ | ---- |
| 字段     | 下拉选择(单选) | 是   | 仅支持json、string数据类型的字段                          | -      | 🟢   |
| 统计规则 | 下拉选择       | 是   | 选择"格式-json校验"，位置在自定义正则上方                 | -      | 🟢   |
| 校验key  | 多选下拉(树形) | 是   | 选择列表为#15696维护的key数据，仅配置过value格式的key可选 | -      | 🟢   |
| 过滤条件 | 选项配置+按钮  | 否   | 点击按钮进行配置                                          | -      | 🟢   |
| 强弱规则 | 单选           | 是   | 强规则/弱规则                                             | 强规则 | 🟢   |
| 规则描述 | 文本输入       | 否   | 自定义描述                                                | -      | 🟢   |

### 校验key选择器（依赖#15696）

| 特性          | 说明                                                           | 来源 |
| ------------- | -------------------------------------------------------------- | ---- |
| 数据来源      | "json格式配置"(#15696)维护的数据列表，取key名                  | 🟢   |
| 可选限制      | 仅配置过value格式信息的key可以被选择，没有配置过的不支持选中   | 🟢   |
| 支持多选      | 可多选key，支持"选择全部"                                      | 🟢   |
| 选择框内搜索  | 支持输入查询                                                   | 🟢   |
| 层级展示      | 按层级展示key结构                                              | 🟢   |
| 默认加载      | 若key数据量多，默认加载前200条展示                             | 🟢   |
| 勾选范围      | 勾选仅对当前层级生效                                           | 🟢   |
| 回显格式      | "key1-key2;key11-key22"                                        | 🟢   |
| 悬浮提示      | 鼠标悬浮展示全部key名信息，默认仅展示前两个                    | 🟢   |
| value格式预览 | 点击"value格式预览"展示弹窗，仅展示勾选key对应的信息，分页展示 | 🟢   |

### value格式预览弹窗

| 列名      | 说明                  | 来源 |
| --------- | --------------------- | ---- |
| key       | key名称               | 🟢   |
| value格式 | 该key对应的正则表达式 | 🟢   |

> 仅展示勾选的key对应的信息，分页展示 🟢
> 悬浮提示内容"校验内容为key名对应的value格式是否符合要求，value格式需要在通用配置模块维护。" 🟢

### 交互逻辑

1. 新增"格式-json格式校验"选项，位置放在自定义正则的上方 🟢
2. json格式校验仅支持数据类型为json、string的字段 🟢
3. 支持doris3.x、sparkthrift2.x、hive2.x数据源 🟢
4. 配置后需要按照层级进行校验，key名需要按照层级匹配是否存在key信息 🟢
5. 需要考虑key数量几千个的情况（性能要求） 🟢
6. 点击"value格式预览"可查看已选key对应的value格式正则 🟢

### 异常处理

| 场景                         | 系统行为                    | 来源 |
| ---------------------------- | --------------------------- | ---- |
| 字段类型非json/string        | 不展示格式-json格式校验选项 | 🟢   |
| #15696中无配置value格式的key | 校验key选择框无可选项       | 🔴   |
| key数据量超大（几千个）      | 默认加载前200条             | 🟢   |

## 页面：结果查询 - 质量报告展示

### 质量报告列表字段

| 列名       | 说明                                                 | 来源 |
| ---------- | ---------------------------------------------------- | ---- |
| 规则类型   | 有效性校验                                           | 🟢   |

| 质检结果   | 校验通过 / 校验不通过                                | 🟢   |

| 详情说明   | 描述具体规则匹配情况                                 | 🟢   |
| 操作       | 校验不通过时可"查看详情"                             | 🟢   |

### 详情说明文案规则

| 质检结果   | 详情说明文案                                             | 来源 |
| ---------- | -------------------------------------------------------- | ---- |
| 校验通过   | 符合规则key为"key1-key2;key11-key22…"时的value格式要求   | 🟢   |
| 校验不通过 | 不符合规则key为"key1-key2;key11-key22…"时的value格式要求 | 🟢   |

### 交互逻辑

1. 校验通过的规则，不记录明细数据 🟢
2. 校验不通过的规则，支持"查看详情" 🟢
3. 校验失败的规则，支持查看日志 🟢
