## 数据资产

### 元数据管理

#### 元数据同步

##### 【149200】验证字段 colExtraAttribute 为空或 columnName 重复时元数据同步正常完成不报错

> 前置条件

```
存在一张包含以下特征列的数据表（可通过数据库直接构造，或使用已知含有此类字段的数据源）：
1. 至少一列的 colExtraAttribute 字段值为 NULL 或空串
2. 触发过至少一次元数据同步（使历史列快照数据存在于 dt_column 相关表中），以便触发新旧列对比逻辑
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 进入【数据资产 > 元数据管理】，找到前置条件中包含空 colExtraAttribute 列的数据源和数据库 | 页面正常展示，数据源和数据库可选 |
| 2 | 选中目标数据库，点击"同步"（或"手动同步"），触发元数据同步任务 | 同步任务提交成功，页面提示操作成功或可在任务列表查看进度 |
| 3 | 进入元数据同步任务列表，等待同步任务执行完成，查看最终状态 | 同步任务状态显示"同步成功"，不出现"同步失败"或错误状态 |
| 4 | 点击同步任务查看详情或日志，确认无 NullPointerException 或 IllegalStateException 相关报错 | 任务日志中无 NPE、无 IllegalStateException，列信息正常写入 |
| 5 | 进入对应数据库的表详情页，查看包含空 colExtraAttribute 列的表字段列表 | 字段列表正常展示，colExtraAttribute 为空的字段显示为空或默认值，不影响其他字段展示 |
