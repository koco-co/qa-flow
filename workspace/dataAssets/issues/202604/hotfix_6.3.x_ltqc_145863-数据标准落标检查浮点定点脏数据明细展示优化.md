## 数据标准

### 落标检查

#### 检查项脏数据明细展示

##### 【145863】验证落标检查脏数据明细中浮点数/定点数字段以完整精度格式展示，不出现科学计数法

> 前置条件

```
1、环境要求：岚图环境已部署 dt-center-metadata 的 hotfix_6.3.x_ltqc_145863 修复版本；测试账号可访问【数据标准】相关菜单。

2、测试表已在 tytest 库下创建完成（含超大浮点、超小浮点、普通小数、整数、NULL 五种场景）：
   表名：ltqc_standard_check_145863
   数据：共 3 行，id=1（超大值 + NULL），id=2（超小值 + NULL），id=3（普通小数 + 1.5）

3、账号要求：具备【数据标准】模块的落标检查新建与查看权限。

4、需提前在【数据标准 > 标准管理 > 标准定义】中存在至少一个已发布的标准目录/标准定义，
   供落标检查任务的"对标标准"字段选择（任意已发布标准即可，无需与测试表字段类型严格匹配）。
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 顶部导航点击【数据标准】，左侧菜单依次展开【标准管理】→ 点击【落标检查】 | 进入落标检查任务列表页，页面顶部有【新建任务】按钮 |
| 2 | 点击【新建任务】，弹出新建任务向导（共 3 步）；当前在第 1 步「检查数据范围」 | 页面显示三步进度条（检查数据范围 / 检查字段列表 / 检查周期），当前高亮第 1 步 |
| 3 | 第 1 步 - 填写「检查数据范围」：<br>① 【数据源】下拉选择已接入的 SparkThrift 数据源（tytest 所在数据源）<br>② 【数据库】下拉选择 `tytest`<br>③ 【数据表】下拉选择 `ltqc_standard_check_145863`<br>④ 分区字段留空（该表无分区）<br>⑤ 【对标标准】区域，点击【标准目录】下拉，选择任意一个已发布的标准目录<br>⑥ 点击右下角【下一步】 | 表单校验通过，进入第 2 步「检查字段列表」；页面左侧展示表的所有字段列表（id / col_float / col_double / col_decimal / col_int / col_null） |
| 4 | 第 2 步 - 配置「检查字段列表」：<br>① 找到字段 `col_int`，将其行右侧的【是否开启检查】开关拨为「开启」<br>② 在 `col_int` 行的【检查项】列点击下拉，选择步骤 3 中所选标准目录下的任意一条检查项<br>③ 点击右下角【临时检查】（立即触发单次检查，无需配置周期） | 弹出确认提示或直接跳转，临时检查任务提交成功；页面提示"临时检查任务创建成功"并返回落标检查任务列表 |
| 5 | 在落标检查任务列表中找到刚创建的 `ltqc_standard_check_145863` 对应任务行，等待【检查状态】列变为「已完成」（可刷新页面） | 任务状态显示「已完成」；【标准达标率】列显示百分比，【不达标字段数/检查失败数】列显示数值（若所有行不达标，则不达标数为 3） |
| 6 | 点击该任务行操作列中的【查看检查结果】 | 跳转到检查结果明细页，列表展示各字段的检查记录，包含列：数据源、数据库、数据表、字段、检查状态、是否达标、检查开始时间、检查结束时间、操作 |
| 7 | 在检查结果列表中，找到字段为 `col_int`、「是否达标」为「未达标」的记录行，点击操作列中的【查看详情】 | 右侧滑出「不达标数据详情」抽屉（标题格式：`col_int - 不达标数据详情`）；抽屉内为表格，展示不达标行的所有字段值，列包含 id / col_float / col_double / col_decimal / col_int / col_null |
| 8 | 核对抽屉中 id=1 行的 `col_double` 列值 | 显示为完整小数字符串（如 `1.2345678901234568E11` 对应修复后应显示 `123456789012.34568`），**不出现科学计数法**（E 符号）；若出现 `E` 则为未修复 |
| 9 | 核对 id=2 行的 `col_double` 列值 | 显示为 `0.000000123456` 或精度等效的完整小数，**不出现** `1.23456E-7` 等科学计数法形式 |
| 10 | 核对 id=1、id=2 行的 `col_decimal` 列值 | id=1 显示为 `1234567890.1234567890`（保留 10 位小数），id=2 显示为 `0.0000001234`，精度完整不截断 |
| 11 | 核对 id=3 行的 `col_float`、`col_double`、`col_decimal` 列值 | 分别显示为 `1.5`、`1.5`、`1.5000000000`，普通小数正常，无多余格式变化 |
| 12 | 核对所有行的 `col_int` 列值 | 分别显示为 `100`、`200`、`300`，整数类型不受影响，无小数点或科学计数法 |
| 13 | 核对 id=1、id=2 行的 `col_null` 列值 | 显示为空（无内容），不报错，页面正常渲染；id=3 行 `col_null` 显示为 `1.5` |
