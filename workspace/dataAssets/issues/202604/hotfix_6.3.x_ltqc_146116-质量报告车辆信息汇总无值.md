## 数据质量

### 数据质量报告

#### 【146116】验证质量报告详情中车辆信息汇总数据正常展示

> 前置条件

已存在已执行过的规则任务（单表监控），且关联了质量报告配置

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 进入【数据质量 > 数据质量报告】 | 质量报告列表正常加载 |
| 2 | 找到目标报告记录，点击【查看】进入详情页 | 成功进入质量报告详情页 |
| 3 | 在详情页找到【车辆信息汇总】模块，查看表格内容 | 车辆信息汇总表格中有数据行，各列（车系、车型、动力类型、车辆数量等）均正常展示，不为空或"--" |

#### 【146116】验证不同租户的质量报告车辆信息汇总数据不互串

> 前置条件

系统中存在至少两个租户，各自有独立的质量报告配置

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 以租户 A 账号登录，进入【数据质量 > 数据质量报告】，点击【查看】打开报告详情 | |
| 2 | 查看【车辆信息汇总】模块中的数据 | 仅展示租户 A 的车辆信息数据 |
| 3 | 退出登录，以租户 B 账号登录，进入相同路径查看报告详情 | |
| 4 | 查看【车辆信息汇总】模块中的数据 | 仅展示租户 B 的车辆信息数据，不包含租户 A 的数据 |

#### 【146116】验证引用标准规则的规则集，过滤条件不被覆盖

> 前置条件

无

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 进入【数据质量 > 规则集管理】，点击【新建规则集】 | |
| 2 | Step 1 填写基础信息：<br>- *选择数据源: SparkThrift2.x<br>- *选择数据库: 已有数据库<br>- *选择数据表: 已有数据表<br>- *规则包名称: 输入规则包名称<br>点击【下一步】 | |
| 3 | Step 2 监控规则页，点击【新增规则】，选择一条内置标准规则，设置【过滤条件】后保存规则集 | 规则集保存成功 |
| 4 | 重新进入该规则集编辑页，查看刚才添加的标准规则的【过滤条件】 | 过滤条件与步骤 3 设置的值一致，未被清空或替换 |
